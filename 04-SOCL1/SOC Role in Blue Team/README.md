## Security Analyst Journey (SOC L1 Fundamentals)

**Objective:** เข้าใจบทบาท หน้าที่ และกระบวนการทำงานพื้นฐานของทีม Security Operations Center (SOC) ในการรับมือกับภัยคุกคามทางไซเบอร์

### 1. SOC Team Structure (โครงสร้างทีม SOC)
การทำงานในฐานะ SOC Analyst ถือเป็น "ปราการด่านหน้า" ที่ต้องทำงานประสานกับสมาชิกในทีมดังนี้:
* **Junior Analyst (L1):** เฝ้าระวังแดชบอร์ด ตรวจสอบ Alerts ขั้นต้น และระงับเหตุเบื้องต้น
* **Senior Analyst (L2/L3):** จัดการกับเคสที่มีความซับซ้อนสูง (Escalated cases) และคอยซัพพอร์ต L1
* **SOC Engineer:** ตั้งค่า ปรับจูน และบำรุงรักษา Security Tools ให้พร้อมใช้งาน
* **SOC Manager:** ดูแลภาพรวมของทีมและรายงานผลลัพธ์ต่อผู้บริหารระดับสูง
* **Incident Responder (IR):** ผู้เชี่ยวชาญพิเศษที่รับผิดชอบจัดการเหตุการณ์โจมตีรุนแรงระดับเมเจอร์ (Major incidents)

### 2. Alert Triage Workflow (กระบวนการตอบสนองขั้นต้น)
จากการจำลองสถานการณ์จริงบนแดชบอร์ด กระบวนการจัดการกับภัยคุกคามมีสเต็ปดังนี้:
1. **Monitor & Identify:** ตรวจสอบ Alert ที่เข้ามา และระบุความผิดปกติ เช่น Malicious IP Address
2. **Escalate:** ในกรณีที่อยู่นอกเหนือขอบเขต ให้ทำการส่งเรื่องต่อ (Escalate) ให้ Senior Analyst ช่วยประเมิน
3. **Containment:** ระงับเหตุเบื้องต้น เช่น การสั่ง Block IP บน Firewall เพื่อตัดการเชื่อมต่อจากฝั่งผู้โจมตีทันที
4. **Resolve:** ตรวจสอบผลลัพธ์ว่าระบบปลอดภัยแล้ว และทำการปิด Ticket (Flag: `THM{until-we-meet-again}`)

*(Note: การเข้าใจกระบวนการทำงานของ Blue Team อย่างลึกซึ้ง ถือเป็นรากฐานสำคัญในการวิเคราะห์ช่องโหว่เพื่อพัฒนาสู่การเป็น Purple Team อย่างเต็มรูปแบบ)*

### 3. Security Hierarchy & Departments (โครงสร้างความปลอดภัยระดับองค์กร)
เป้าหมายด้านความปลอดภัยของแต่ละบริษัทจะแตกต่างกันไปตามประเภทธุรกิจ (Business Objectives) ดังนั้น ผู้บริหารระดับสูงอย่าง CEO จึงต้องมีบทบาทสำคัญในการกำหนดทิศทาง โดยส่งมอบหมายให้ **CISO (Chief Information Security Officer)** เป็นผู้ตัดสินใจหลักในการวางโครงสร้างและกลยุทธ์ทางไซเบอร์

ในองค์กรขนาดใหญ่ CISO จะดูแลแผนกความปลอดภัยเฉพาะทางต่างๆ ดังนี้:
* **Blue Team (Defensive Security):** ทีมป้องกันและตั้งรับภัยคุกคาม (เช่น SOC Analysts, Incident Responders, Security Engineers) ทำหน้าที่เฝ้าระวังและตอบสนองต่อการโจมตี
* **Red Team (Offensive Security):** ทีมจำลองการโจมตี (เช่น Pentesters, Ethical Hackers) ทำหน้าที่ค้นหาช่องโหว่และทดสอบความแข็งแกร่งของระบบ
* **GRC Team (Governance, Risk, and Compliance):** ทีมจัดการนโยบาย ประเมินความเสี่ยง และดูแลให้องค์กรปฏิบัติตามมาตรฐานความปลอดภัย (เช่น PCI DSS)

*(Analytical Note: การผสานองค์ความรู้ระหว่างยุทธวิธีป้องกันของ Blue Team และเทคนิคเจาะระบบของ Red Team เข้าด้วยกัน จะนำไปสู่แนวคิดแบบ Purple Team ซึ่งช่วยยกระดับความปลอดภัยขององค์กรได้อย่างมีประสิทธิภาพสูงสุด)*

### 4. Blue Team Departments (เจาะลึกโครงสร้างหน่วยตั้งรับ)
**Blue Team** คือทีมที่มุ่งเน้นด้าน **Defensive Security** ทำหน้าที่เฝ้าระวัง แจ้งเตือน และตอบสนองต่อการโจมตีอย่างรวดเร็ว โดยภายใน Blue Team สามารถแบ่งเป็นหน่วยย่อยตามระดับความรับผิดชอบได้ดังนี้:

* **Security Operations Center (SOC):** ศูนย์กลางการป้องกันด่านแรกขององค์กร ประกอบด้วย
  * **L1 Analysts:** คัดกรอง Alerts เบื้องต้น และจัดการเหตุการณ์ระดับเริ่มต้น
  * **L2 Analysts:** วิเคราะห์และสืบสวนการโจมตีที่มีความซับซ้อนสูงขึ้น
  * **Engineers:** ติดตั้งและดูแลระบบรักษาความปลอดภัย (เช่น EDR, SIEM)
  * **Manager:** บริหารจัดการทีม SOC ในภาพรวม
* **CIRT (Cyber Incident Response Team):** หรือทีม "นักดับเพลิงไซเบอร์" (บางครั้งเรียกว่า CSIRT หรือ CERT) เป็นหน่วยตอบสนองเหตุฉุกเฉินที่จะเข้ามาจัดการเมื่อเกิดเหตุการณ์โจมตีรุนแรงที่อยู่เหนือการควบคุมของ SOC 
* **Specialized Defensive Roles:** หน่วยผู้เชี่ยวชาญเฉพาะทางสำหรับภัยคุกคามเชิงลึก เช่น:
  * **Digital Forensics Analyst:** ผู้เชี่ยวชาญการกู้คืนและแกะรอยหลักฐานดิจิทัลจากหน่วยความจำ
  * **Threat Intelligence Analyst:** ผู้รวบรวมและวิเคราะห์ข้อมูลข่าวกรองของกลุ่มแฮกเกอร์
  * **AppSec Engineer:** ผู้เชี่ยวชาญด้านความปลอดภัยของการพัฒนาซอฟต์แวร์
  * **AI Researcher:** ผู้ศึกษาวิธีการป้องกันภัยคุกคามที่เกิดจาก AI

  ### 5. SOC Career Path & Work Environments (เส้นทางอาชีพและสภาพแวดล้อมการทำงาน)
การเริ่มต้นในตำแหน่ง SOC L1 ถือเป็นก้าวแรกที่ยอดเยี่ยมในการเก็บเกี่ยวประสบการณ์จริงจากการรับมือการโจมตี โดยรูปแบบการทำงานของทีม SOC จะแบ่งออกเป็น 2 สภาพแวดล้อมหลักที่แตกต่างกันอย่างชัดเจน ได้แก่:

* **Internal SOC (ทีมความปลอดภัยประจำองค์กร):**
  * ทำหน้าที่ปกป้องระบบขององค์กรต้นสังกัดเพียงแห่งเดียว (เช่น SOC ของธนาคาร)
  * สภาพแวดล้อมมีความกดดันน้อยกว่า มีเวลาโฟกัสและวิเคราะห์ข้อมูลได้ลึก
  * เชี่ยวชาญการใช้ Security Tools เฉพาะทางเพียงไม่กี่ตัว
  * โอกาสพบเจอการโจมตีระดับ Major Incidents ของจริงมีค่อนข้างน้อย

* **MSSP - Managed Security Services Provider (ผู้ให้บริการด้านความปลอดภัย):**
  * ทำหน้าที่ดูแลและปกป้องระบบให้บริษัทลูกค้าหลายสิบแห่งพร้อมกัน
  * สภาพแวดล้อมการทำงานรวดเร็ว กดดันสูง ต้องจัดการกับคิว Alerts ที่เข้ามาตลอดเวลา
  * ต้องมีความยืดหยุ่นในการเรียนรู้และใช้งาน Security Tools ที่หลากหลายมาก
  * พบเจอการโจมตีและช่องโหว่ของจริงแทบทุกสัปดาห์ ทำให้เก็บเกี่ยวประสบการณ์ได้รวดเร็วมาก

**Next Steps for SOC L1:**
หลังจากเก็บเกี่ยวประสบการณ์ในฐานะด่านหน้า (L1) เรียบร้อยแล้ว เส้นทางการเติบโตสามารถขยับขยายได้หลายสาย เช่น เลื่อนขั้นเป็น **SOC L2 Analyst**, เปลี่ยนสายไปดูแลระบบ (SOC Engineer), ย้ายไปทีมรับมือเหตุฉุกเฉิน (CIRT), หรือมุ่งสู่สายบริหารเพื่อเป็น CISO 

### 6. Security Operations Case Study: Role-Based Incident Response
จากการทดสอบบทบาทสมมติในฐานะ **CISO (Chief Information Security Officer)** เพื่อบริหารจัดการ 7 เหตุการณ์วิกฤตที่เกิดขึ้นพร้อมกัน สามารถสรุปหลักการจัดสรรทรัพยากรบุคคลตามความเชี่ยวชาญ (Specialization) ได้ดังนี้:

| เหตุการณ์ (Scenario) | ผู้รับผิดชอบ (Primary Role) | หน้าที่สำคัญ (Key Action) |
| :--- | :--- | :--- |
| ตรวจพบความผิดปกติเบื้องต้นจากระบบ SIEM | **SOC L1 Analyst** | ทำการคัดกรอง (Triage) และประเมินสถานการณ์ขั้นต้น |
| พนักงานเผลอรันไฟล์มัลแวร์จากอีเมล Phishing | **SOC L2 Analyst** | วิเคราะห์มัลแวร์เชิงลึก (Deep Analysis) และสืบสวนหาที่มา |
| ระบบหลักของสาขาในต่างประเทศโดน Ransomware | **CIRT / CSIRT** | เข้าควบคุมสถานการณ์และตอบสนองเหตุฉุกเฉินทันที |
| ต้องการตรวจสอบมาตรฐานความปลอดภัยข้อมูลบัตรเครดิต | **GRC Auditor** | ตรวจสอบการปฏิบัติตามข้อกำหนดทางกฎหมาย (PCI DSS) |
| ตรวจสอบช่องโหว่ของเว็บไซต์เวอร์ชันใหม่ก่อนเปิดตัว | **Penetration Tester** | ค้นหาจุดอ่อน (Vulnerabilities) ของระบบด้วยวิธี Offensive |
| ระบบจัดเก็บข้อมูลของ SIEM เต็มจนใช้งานไม่ได้ | **SOC Engineer** | ซ่อมแซม บำรุงรักษา และตั้งค่าระบบโครงสร้างพื้นฐาน |
| มีข่าวกรองว่ากลุ่มแฮกเกอร์ FIN7 กำลังมุ่งเป้ามาที่บริษัท | **Threat Intelligence Analyst** | วิเคราะห์กลยุทธ์ (Tactics) และเตรียมแผนป้องกันเชิงรุก |

**Analytical Conclusion:**
หัวใจสำคัญของทีม Blue Team ที่มีประสิทธิภาพไม่ได้ขึ้นอยู่กับเครื่องมือที่ทันสมัยเพียงอย่างเดียว แต่ขึ้นอยู่กับการประสานงานระหว่าง "ด่านหน้า" (SOC L1) ที่รวดเร็ว และ "ผู้เชี่ยวชาญเฉพาะทาง" (Specialized Roles) ที่แม่นยำ เพื่อสร้างเกราะป้องกันแบบหลายชั้น (Defense in Depth) ให้แก่องค์กร
