# SOC L1 Operations: Triage, Reporting, and Escalation

**Objective:** สรุปกระบวนการทำงานมาตรฐานสำหรับนักวิเคราะห์ความปลอดภัยทางไซเบอร์ระดับที่ 1 (SOC L1 Analyst) ครอบคลุมตั้งแต่การตรวจสอบการแจ้งเตือน (Alert Triage) การเขียนรายงาน (Reporting) และการยกระดับเหตุการณ์ (Escalation)

---

## 1. Alert Triage (การคัดกรองการแจ้งเตือน)
กระบวนการแรกเมื่อได้รับ Alert จากระบบ SIEM หรือ EDR เพื่อวิเคราะห์ความน่าเชื่อถือของภัยคุกคาม

* **Initial Action:** เข้าควบคุมเคสโดยเปลี่ยนสถานะ Alert จาก `New` เป็น `In Progress`
* **Investigation:** ตรวจสอบ Logs อย่างละเอียดเพื่อหาบริบทของการโจมตี (เป้าหมายคือใคร, พฤติกรรมเป็นอย่างไร)
* **Verdict (การตัดสินใจ):**
  * **True Positive:** เป็นการโจมตีหรือภัยคุกคามจริง (ต้องดำเนินการต่อหรือส่งต่อ L2)
  * **False Positive:** เป็นการแจ้งเตือนผิดพลาดที่เกิดจากการทำงานปกติของระบบ (ทำการปิดเคส `Closed`)

## 2. Alert Reporting (การเขียนรายงาน)
เมื่อพบ True Positive หรือเหตุการณ์ที่ต้องส่งต่อ L1 จะต้องเขียนรายงานสรุป (Analyst Comment) เพื่อให้ผู้ที่รับช่วงต่อเข้าใจสถานการณ์ได้อย่างรวดเร็ว โดยใช้หลักการ **The Five Ws**:

* **Who:** ใครคือผู้กระทำ หรือใครคือเหยื่อ (เช่น ชื่อผู้ใช้, อีเมล)
* **What:** เกิดการกระทำหรือเหตุการณ์อะไรขึ้น (เช่น ดาวน์โหลดไฟล์มัลแวร์, ตรวจพบ Phishing)
* **When:** เหตุการณ์เริ่มต้นและสิ้นสุดเมื่อใด (Timestamp)
* **Where:** เหตุการณ์เกิดขึ้นที่ไหน (เช่น IP Address, เครื่อง Workstation, ระบบ Network)
* **Why:** เหตุผลที่ตัดสินใจว่าเคสนี้เป็น True Positive หรือเหตุใดจึงน่าสงสัย

## 3. Alert Escalation (การยกระดับและส่งต่อเหตุการณ์)
หาก Alert มีความซับซ้อนหรือรุนแรง L1 จะต้องส่งเคสต่อให้ L2 Analyst (Escalation) สถานการณ์ที่ควรพิจารณาส่งต่อ ได้แก่:
* ตรวจพบการโจมตีรุนแรงที่ต้องใช้การสืบสวนเชิงลึก (DFIR)
* ต้องการการตอบสนองทันที เช่น การแยกเครื่องออกจากเครือข่าย (Host Isolation) หรือลบมัลแวร์
* มีความจำเป็นต้องสื่อสารกับบุคคลภายนอก หรือผู้บริหาร
* L1 ไม่เข้าใจพฤติกรรมของ Alert นั้นๆ อย่างถ่องแท้ (ไม่ควรปิดเคสหากไม่มั่นใจ)

## 4. Emergency Communication Best Practices (แนวทางการสื่อสารในภาวะฉุกเฉิน)
* **Unresponsive L2:** หากมีเคสวิกฤตแต่ L2 ไม่ตอบสนอง ให้ติดต่อตามลำดับขั้น: `L2` -> `L3` -> `Manager`
* **Compromised Accounts:** ห้ามติดต่อเหยื่อผ่านช่องทางที่ถูกแฮก (เช่น ถ้า Slack โดนแฮก ให้โทรศัพท์แทน)
* **Misclassification:** หากพบว่าตนเองวิเคราะห์ Alert ผิดพลาดและอาจปล่อยผ่านภัยคุกคามไป ต้อง **แจ้ง L2 ทันที** เพราะผู้โจมตีอาจกำลังแฝงตัวอยู่ในระบบ
* **Broken Logs/SIEM:** หาก Logs ไม่สามารถอ่านได้ ห้ามข้าม Alert นั้น แต่ให้สืบสวนเท่าที่ทำได้และแจ้งปัญหาให้ Engineer ทราบ
